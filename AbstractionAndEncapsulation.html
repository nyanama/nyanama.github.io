Difference Between Abstraction and Encapsulation

Before going to the differences, first we will understand what is Abstration and what is Encapsulation

Abstraction

The way of projecting the essential things and hidding the complex things is known as Abstarction.

In other words Abstraction is a process of hidding implementation details and showing only the functionality.

Abstraction can be achieved by using interfaces and abstract classes.

Encapsulation

It is a process of wrapping similar group of things into single unit.

And also it is a process of binding data and methods to hide the data using access modifiers (this is called data hidding).

Complete encapsulation in java is achieved by making variables of a class private and access to them, outside the class, is provided only through public methods (getters and setters in this case).

Differences

Abstraction Encapsulation

1. Abstraction solves the problem in design level.

1. Encapsulation solves the problem in implementation level.

2. Abstraction is used to hide the unwanted things and showcasing the relavent things.

2. Encapsulation is used to binding data and code into a single unit to protect from outside world.

3. Abstraction lets you focus on what the object does instead of how it does it.

3. Encapsulation means hidding the internal details of how an object does something.
