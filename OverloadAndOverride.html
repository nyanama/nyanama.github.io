Difference Between Method Overloading and Overriding

Differences

Overloading Overriding

Creating the multiple methods in a class with same name and different signature.

Creating a method in parent and child class with same name and signature.

Method names should be the same with different arguments.

Method names and arguments should be the same.

Return type can be different (we can change the return type).

Return type can be same or a co variant (we can not have different return type exept co varian).

It increases the redability of the code.

To override the existing implementation of the parent class.

Binds the appropriate methods at comiletime.

Binds the appropriate methods at runtime.

Static methods can be overloaded, as this is compile time binding (static methods also binds at compiletime).

Can not override static methods as overriding is runtime and static methods are at compiletime.

	public static void main(String[] args) {
    	OverloadingExample test = new OverloadingExample();

	}
	public String methodOne(String str) {
		return "String";
	}
	public String methodOne(Object str) {
		return "Object";
	}
}

Answwer is "String"
Reason: If more than one method is applicable and accesable to a method innvocation then comilar will give preference to subclass arguments. Here object is super class and String is sub class so sub class argument method called and returned String.

public class OverloadingExample {
	public static void main(String[] args) {
    	OverloadingExample test = new OverloadingExample();
		System.out.println(test.methodOne(null));
	}
	public String methodOne(String str) {
		return "String";
	}
	public String methodOne(Object str) {
		return "Object";
	}
	public String methodOne(Integer str) {
		return "Object";
	}

}

Answwer is - Compile time error - The method methodOne(String) is ambiguous for the type Test
Reason: As we discussed if multiple methods are eligible to access a method call it will give preference to the method with sub class arguments but here we have two methods with sub class arguments String and Integer. So compiler can not decide which method to invoke and gives an error.

What is co-variant method overriding?

Before Java 1.5 we have a hard rule saying that for method overriding the return type should be same for parent and child class methods. But from 1.5 the restriction is relaxed. Now we can have same return type or subclass of the return type of parent class.
